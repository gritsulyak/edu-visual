What is Consistent Hashing?

Consistent hashing maps both servers and data keys onto the same numeric space (a "ring"). Each key is owned by the first server whose position is ≥ the key's position — going clockwise around the ring.

1

Hash the server name → place it at that position on the ring.

2

Hash each key → place the key on the ring at its position.

3

Walk clockwise from the key's position until you find the next server node. That server owns the key.

4

When a server is removed, only the keys between it and the previous server are remapped — not the entire dataset.

Virtual Nodes (vnodes)

A single physical server is placed on the ring multiple times using different identifiers (e.g. S1#0, S1#1, S1#2). Each is a virtual node.

This solves two problems:

① Even distribution — without vnodes, servers cluster unevenly. With vnodes, load spreads more uniformly.

② Weighted capacity — a server with twice the RAM gets twice the vnodes, and therefore receives roughly twice the keys.

Change the Virtual nodes per server slider in the header and watch how the ring table and key assignments change.

Adding & Removing Servers

Adding a server: its vnodes are inserted into the sorted ring. For every new vnode, the keys that were previously owned by the next clockwise server but now fall between a predecessor and the new vnode are reassigned. Only a fraction of total keys move.

Removing a server: its vnodes are removed. Keys owned by those vnodes walk further clockwise to the next surviving node. Again — only the affected keys move.

Try it: load the demo, then remove a server with the × button. Watch which keys change their assignment in the table.

How the Position is Calculated

This visualizer uses FNV-1a 32-bit hash, mapped onto a ring of 1,000,000 positions:

// FNV-1a 32-bit
offset_basis = 2166136261
prime = 16777619
hash = offset_basis
for each byte b in input:
  hash = (hash XOR b) * prime // mod 2³²
position = hash mod 1_000_000
// Server vnode key:
input = "ServerName#vnodeIndex"
// e.g. "S1#0", "S1#1", "S1#2"
// Data key:
input = keyName // e.g. "user:42"

The raw 32-bit hash value and the final position (mod 1M) are both shown in the ring table so you can verify every step.
